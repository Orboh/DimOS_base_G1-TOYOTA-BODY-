# 1. 今回のゴール: G1 でカメラを DimOS 上で使えるようにする

作成日: 2026-06-01

## 背景

2026-05-20 の調査では、G1 実機で DimOS agentic stack を起動したときに、
カメラ画像が使えない問題と LiDAR/点群が使えない問題を調査した。

前回の結果として、LiDAR 側は Mid-360 LiDAR / FastLIO 経路を
`unitree-g1-agentic` に統合し、DimOS Viewer 上で点群・地図・odometry が
流れるところまで確認できた。

一方で、カメラ側は `G1Connection` に WebRTC 映像を DimOS stream へ流す処理を
追加したものの、G1 との WebRTC 接続が最後まで確立せず、実機映像が
`/color_image` に流れるかどうかを確認できていない。

前回メモ:

```text
yokote/20260520/実施メモ/３．抱えている問題.md
```

## 今回の目的

今回の目的は、前回解決できなかった WebRTC 接続問題を解決し、
G1 実機のカメラ映像を DimOS 上で利用できる状態にすること。

最終的には、G1 で LiDAR だけでなくカメラも使える状態にし、
視覚認識・空間認識・ナビゲーション系の skill が必要なセンサ入力を
受け取れるようにする。

## 現在わかっている状態

前回までに確認済みのこと:

- G1 main computer: `192.168.123.161`
- G1 Jetson: `192.168.123.164`
- Mid-360 LiDAR: `192.168.123.120`
- PC 有線 IP: `192.168.123.212`
- G1 の `tcp/8081` は疎通可能
- LiDAR / FastLIO は `unitree-g1-agentic` に統合済み
- `SpatialMemory` の CLIP モデル不足は解消済み
- `G1Connection` には `color_image` と `camera_info` の stream を追加済み
- `G1Connection` は `self.connection.video_stream()` を `color_image.publish` に接続済み

前回未解決のこと:

- WebRTC の SDP answer 受信と `setRemoteDescription` までは到達する
- その後の Data Channel Validation が完了しない
- そのため `[webrtc_fix] datachannel open!` が出ない
- 結果として、G1 本体カメラの映像が `/color_image` に流れるか未確認

## 今回やること

1. G1 との WebRTC 接続ログを再確認する
2. `wait_datachannel_open` の Validation handshake が止まる原因を調べる
3. `[webrtc_fix] datachannel open!` が出る状態にする
4. `Data Channel Verification: OK` と `Peer Connection State: connected` を確認する
5. WebRTC 接続後に `/color_image` に G1 カメラ画像が流れるか確認する
6. DimOS Viewer / Foxglove などの可視化画面でカメラ画像が黒画面ではなく表示されるか確認する
7. 必要なら `G1Connection` の camera stream / camera_info / blueprint 配線を修正する

## 成功条件

今回の成功条件は以下。

- `unitree-g1-agentic` が G1 実機に対して WebRTC 接続を確立できる
- ログに以下が出る

```text
[webrtc_fix] datachannel open!
Data Channel Verification: OK
Peer Connection State: connected
```

- `/color_image` topic に G1 実機カメラの画像が流れる
- `/camera_info` topic に camera info が流れる
- 可視化画面で G1 カメラ画像が確認できる
- LiDAR / FastLIO 側の動作を壊さず、`/lidar`, `/global_map`, `/odometry` なども引き続き確認できる

## 起動コマンド

前回の続きとして、まずは Docker ではなくホスト側 venv で起動する。

理由:

- 前回の暫定修正は `dimensional-applications/.venv` 配下の
  `unitree_webrtc_connect` に入っている
- Docker image 内に同じ修正が入っている保証がない
- `dimensional-applications-g1/.venv` は `../dimensional-applications/.venv` への symlink

起動コマンド:

```bash
cd /home/rad907/workspace/dimos-go2
LIDAR_HOST_IP=192.168.123.212 LIDAR_IP=192.168.123.120 \
  ./dimensional-applications-g1/run_robot.sh \
  --disable spatial-memory \
  --disable object-tracking
```

確認したいログ:

```text
[webrtc_fix] ICE complete, candidates=..., sending SDP to peer...
[webrtc_fix] Got SDP answer from peer (len=...)
[webrtc_fix] setRemoteDescription...
[webrtc_fix] waiting for datachannel...
[webrtc_fix] datachannel open!
Data Channel Verification: OK
Peer Connection State: connected
```
